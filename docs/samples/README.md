# Sample import files

## `valspar_rebuy_sample_round2.xlsx` (Valspar pool + real board)

Generated by:

```bash
python3 scripts/build_valspar_rebuy_sample.py
```

- **Entries:** every row from `valspar_entries_25.csv` (participant + six pros exactly as in that file).
- **Leaderboard:** `Slash Golf Jsons/leaderboards.json` (same Valspar / Innisbrook field; includes **R2 cuts**).  
  *A PDF export isn’t machine-readable in Cursor, so this JSON is the stand-in for “current leaderboard.”*
- **Logic:** entries are sorted by how weak their six picks are **on that board** (CUT = worst). Weaker entries get **more** replace pairs; the **Replace** column uses the **same spelling as your CSV**; **Replace with** uses **leaderboard/API spelling** (e.g. `Xander Schauffele`, `Nicolai Højgaard`) so you exercise name matching.
- **Typos handled in the script** (for ranking only): `Schuaffele`→Schauffele, `Speith`→Spieth, `Nicolai Hojgaard`→Højgaard.

Re-run the script after you refresh `leaderboards.json` from the API if you want the sheet to match a newer round.

## `rebuy_example_mix.xlsx`

**The file in this folder is only a placeholder** unless you regenerate it. It does **not** automatically know your live database or the real Masters leaderboard.

### Generate a real “tournament week” rebuy sheet

Run the script against the **same database** your API uses (`backend/.env` → `DATABASE_URL`):

```bash
cd /path/to/masters
pip install -r backend/requirements.txt
python3 scripts/build_rebuy_example_xlsx.py
```

Optional:

```bash
# Specific tournament
python3 scripts/build_rebuy_example_xlsx.py --tournament-id 1

# Prefer leaderboard snapshot saved for round 2 (falls back to latest if none)
python3 scripts/build_rebuy_example_xlsx.py --prefer-round 2

# Custom output path
python3 scripts/build_rebuy_example_xlsx.py --output ~/Desktop/rebuys_round2.xlsx
```

### What the script does

1. Picks a tournament (by `--tournament-id`, or the one whose `status` looks like “in progress”, else the most recent by `start_date`).
2. Loads **every entry** in that tournament: **Player Name** = participant name, **Professional 1–6** = the six golfer names from your `players` table (fallback: names from the snapshot).
3. Reads the **latest `score_snapshots` row** for that tournament, optionally filtered by `--prefer-round` (defaults to `tournament.current_round`).
4. Sorts the leaderboard by **position** (CUT/WD/etc. treated as very far back).
5. Ranks pool entries by how weak their six picks are **on that board** (average position).
6. Builds rebuys like a normal week:
   - **Weaker** entries: more **Replace / Replace with** pairs; originals are their **worst-position** golfers; replacements are the **top of the board** who are **not already** on that entry.
   - **Stronger** entries: fewer swaps; roughly **every third** of the strongest entries gets **no** rebuys (weekend loyalty).

### Requirements

- Entries already imported for that tournament.
- At least one **leaderboard** saved in `score_snapshots` (run your normal **sync** from admin so the board matches what you see in the app).

### Duplicate participant names

If two entries share the same participant name, the importer only attaches rebuys to the **first** match. The script prints a **warning** if it detects duplicate names.
